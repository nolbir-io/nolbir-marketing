# Contributing

Contribution guide for the Mamurify marketing strategy repository.

## Principles

- Keep strategy and operations in sync: every strategy decision should show up in execution docs.
- Prefer small, frequent updates over large rewrites.
- Record rationale for material changes; avoid silent edits to key assumptions.

## Ownership and Reviews

- Every active document must include a single DRI in its metadata section.
- Review SLA for requested document feedback: 2 business days.
- Strategic docs (`docs/playbook/`, `docs/plan-6m/`) require Marketing Lead review.

## Change Process

1. Update relevant artifact(s) in functional folders.
2. Reflect decision updates in `docs/playbook/open-questions-register.md` when assumptions change.
3. If month priorities shift, update `docs/plan-6m/month-0X.md` in the same change.
4. Keep changelog rows where present; do not remove historical context.

## Definition of Done for New Marketing Docs

- Objective and intended audience are explicit.
- Owner and review cadence are included.
- Success metric is named and measurable.
- Dependencies and next action are clear.

## Naming Conventions

- Use kebab-case for filenames.
- Keep docs focused: one document, one operational purpose.
- Place recurring templates in the nearest workstream folder.
