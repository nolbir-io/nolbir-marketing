# Own Playbook — The Road to 1,000 Merchants

A Go-to-Market playbook for building e-commerce infrastructure in Uzbekistan, organized by milestone phase. Read in order.

> **Status:** Working scaffold. Phase definitions are a default milestone structure (10 → 100 → 1,000), to be revised once real numbers, ICP, and timeline are agreed.

## How to use this playbook

Each phase file answers the same five questions:

1. **Goal** — what "done" looks like at the end of this phase.
2. **ICP focus** — which slice of the merchant universe we're targeting *in this phase only*.
3. **Motion** — how we acquire and activate (founder-led, partner-led, inbound, field).
4. **Channels & content** — what we run and what we publish.
5. **Exit criteria** — the metric and qualitative signals that say "move to the next phase."

The marketing-skill folders at the repo root (`brand-voice/`, `campaigns/`, `competitive-analysis/`, etc.) are the *workshops* — playbook phases pull from them and push back into them.

## Top-level unknowns

Things that affect every phase below and need to be decided once:

- [unknown] **North-star metric.** Is "1,000 merchants" measured at *signed*, *live*, or *transacting*?
- [unknown] **Timeline.** Over what horizon — 12 months? 24? 36?
- [unknown] **Geographic scope.** Tashkent first, or nationwide from day one?
- [unknown] **Language posture.** Default to Uzbek (Latin / Cyrillic?) or Russian for first-touch?
- [unknown] **Founder bandwidth.** How much of the early motion is founder-led vs. delegated?
- [unknown] **Capital envelope.** What's the marketing/sales budget per phase?

## Review loop into this playbook

- Every weekly and monthly decision that changes assumptions must be recorded in [open-questions-register.md](open-questions-register.md).
- At the end of each month, update phase-level assumptions in the relevant phase doc.
- At the end of each quarter, adjust phase exit criteria only with explicit rationale.
